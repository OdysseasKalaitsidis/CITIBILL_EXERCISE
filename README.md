# Travel Planner

Εφαρμογή οργάνωσης τριήμερου ταξιδιού. Ο χρήστης επιλέγει
δραστηριότητες και τις τοποθετεί στις μέρες του ταξιδιού με
συγκεκριμένες ώρες.

## Εργαλεία

- React 19 + TypeScript
- Vite
- Tailwind CSS
- Lucide React

## Εκτέλεση

```
npm install
```

```
npm run dev
```

## Λειτουργίες

- Αναζήτηση και φιλτράρισμα δραστηριοτήτων ανά tag
- Ταξινόμηση ανά τίτλο και κόστος
- Drag & drop δραστηριοτήτων στις μέρες
- Επεξεργασία ωραρίου inline
- Ανίχνευση επικαλύψεων ωραρίου
- Συνολικό κόστος ανά ημέρα και για το ταξίδι
- Light / Dark mode
- Αποθήκευση στο localStorage

## Υποθέσεις

- Το κόστος είναι σταθερό ανά δραστηριότητα (όχι ανά ώρα ή ανά άτομο).
- Το νόμισμα όλων των δραστηριοτήτων είναι το Ευρώ.
- Δεν επιβάλλεται ανώτατο όριο συνολικών ωρών ανά ημέρα, το όριο των 7 ωρών ισχύει αποκλειστικά ανά δραστηριότητα.
- Τα tags από το αρχείο JSON εμφανίζονται για τη διευκόλυνση του φιλτραρίσματος στο UI, παρόλο που δεν αναφέρονται στις προδιαγραφές.
- Μια δραστηριότητα δε μπορεί να εκτείνεται σε δύο μέρες.
- Επικαλύψεις ωραρίου μεταξύ δραστηριοτήτων της ίδιας ημέρας δεν επιτρέπονται.
- Η γλώσσα της εφαρμογής ακολουθεί τη γλώσσα περιγραφής της άσκησης δηλαδή Ελληνικά.
- Η εφαρμογή θα χρησιμοποιηθεί σε web και mobile περιβάλλον
- Η εφαρμογή αφορά τον προγραμματισμό δραστηριοτήτων στην τοπική ώρα του προορισμού.

## Ανοιχτά Θέματα

- Είναι το κόστος ανά δραστηριότητα, ανά ώρα ή ανά άτομο; Τι συμβαίνει εάν υπάρχουν διαφορετικές επιλογές (π.χ. μουσείο με ή χωρίς ξεναγό);
- Μπορούν οι δραστηριότητες να έχουν διαφορετικά νομίσματα; Εάν ναι, πώς υπολογίζεται το ημερήσιο σύνολο (σταθερή ισοτιμία ή ζωντανή ισοτιμία);
- Ποιο είναι το έγκυρο χρονικό εύρος μιας ημέρας (8, 16 ή 24 ώρες); Υπάρχει μέγιστη συνολική διάρκεια δραστηριοτήτων ανά ημέρα;
- Μπορεί μια δραστηριότητα να ξεκινά την Ημέρα 1 και να ολοκληρώνεται την Ημέρα 2, εφόσον είναι κάτω από 7 ώρες;
