# Travel Planner

Εφαρμογή οργάνωσης τριήμερου ταξιδιού. Ο χρήστης επιλέγει δραστηριότητες και τις τοποθετεί στις μέρες του ταξιδιού με συγκεκριμένες ώρες.

## Εργαλεία

- React 19 + TypeScript
- Vite
- Tailwind CSS
- Lucide React

## Εκτέλεση

```bash
npm install
npm run dev
```

## Λειτουργίες

- Αναζήτηση και φιλτράρισμα δραστηριοτήτων ανά tag
- Ταξινόμηση ανά τίτλο και κόστος
- Drag & drop δραστηριοτήτων στις μέρες
- Επεξεργασία ωραρίου inline
- Ανίχνευση και αποκλεισμός επικαλύψεων ωραρίου
- Συνολικό κόστος ανά ημέρα και για το ταξίδι
- Light / Dark mode
- Αποθήκευση στο localStorage

## Παραδοχές

- Το κόστος είναι σταθερό ανά δραστηριότητα, όχι ανά ώρα ή ανά άτομο.
- Το νόμισμα είναι το Ευρώ (EUR) για όλες τις δραστηριότητες.
- Μία «ημέρα» δεν έχει επιβεβλημένο όριο συνολικών ωρών — το όριο των 7 ωρών ισχύει αποκλειστικά ανά δραστηριότητα.
- Οι δραστηριότητες δεν μπορούν να εκτείνονται μετά τα μεσάνυχτα (η έναρξη και η λήξη πρέπει να βρίσκονται εντός της ίδιας ημερολογιακής ημέρας).
- Κάθε ημέρα αντιστοιχεί σε μία σταθερή ημερομηνία (Ημέρα 1 = Ημερομηνία Χ κ.λπ.), με την ημερομηνία έναρξης του ταξιδιού να επιλέγεται από τον χρήστη.
- Τα tags από το αρχείο JSON εμφανίζονται για τη διευκόλυνση του φιλτραρίσματος στο UI, παρόλο που δεν αναφέρονται ρητά στις προδιαγραφές.
- Η γλώσσα της διεπαφής (UI) ακολουθεί τη γλώσσα περιγραφής της άσκησης (Ελληνικά).
- Εφαρμόζεται πλήρης αποκλεισμός (validation) επικάλυψης δραστηριοτήτων την ίδια ημέρα (τόσο κατά την προσθήκη από το modal όσο και κατά την inline τροποποίηση του ωραρίου).

## Ανοιχτά Θέματα προς Διευκρίνιση

- Είναι το κόστος ανά δραστηριότητα, ανά ώρα ή ανά άτομο; Τι συμβαίνει εάν υπάρχουν διαφορετικές επιλογές (π.χ. μουσείο με ή χωρίς ξεναγό);
- Μπορούν οι δραστηριότητες να έχουν διαφορετικά νομίσματα; Εάν ναι, πώς πρέπει να υπολογίζεται το ημερήσιο σύνολο — με σταθερή ισοτιμία, με ζωντανή ισοτιμία ή να εμφανίζονται ξεχωριστά;
- Ποιο είναι το έγκυρο χρονικό εύρος μιας ημέρας — 8, 16 ή 24 ώρες; Υπάρχει μέγιστη συνολική διάρκεια δραστηριοτήτων ανά ημέρα;
- Μπορεί μια δραστηριότητα να ξεκινά την Ημέρα 1 και να ολοκληρώνεται την Ημέρα 2, εφόσον η συνολική της διάρκεια είναι κάτω από 7 ώρες;
- Θα πρέπει οι ημερομηνίες του ταξιδιού να είναι σταθερές ή να ορίζονται από τον χρήστη;
- Τι συμβαίνει εάν δύο δραστηριότητες συμπίπτουν χρονικά την ίδια ημέρα — θεωρείται σφάλμα (και εμποδίζεται η καταχώρηση) ή επιτρέπεται;
- Η απαίτηση για τη γλώσσα του UI είναι στα Ελληνικά (όπως η περιγραφή) ή στα Αγγλικά;
- Είναι απαραίτητη η υποστήριξη της εφαρμογής σε κινητές συσκευές (mobile responsiveness);
